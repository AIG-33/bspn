# План запуска интернет-ресурса БСПН для привлечения новых членов

Документ для презентации Правлению. Описывает: зачем сайт устроен именно так,
какие флоу проходит посетитель, что уже готово в демо-версии и какие этапы
ведут от демо к работающему инструменту привлечения членов.

---

## 1. Цель и логика ресурса

**Цель:** новые члены союза и своевременная уплата взносов.

**Кто наш посетитель:** руководитель компании с конкретной проблемой
(проверка, спорная норма, потребитель-экстремист, спор с контрагентом),
который ищет поддержку и экспертизу. Он не читает миссии — он ищет ответ
на вопрос «что я получу и сколько это стоит».

**Принцип всего сайта:** посетитель должен получить первую пользу за
10–15 секунд, узнать свою проблему — и только потом мы предлагаем вступить.
Каждая страница отвечает на три вопроса: *что болит → как союз решает →
что сделать прямо сейчас*.

## 3. Все флоу посетителя (что рассказывать на собрании)

**Флоу А — «холодный» посетитель с проблемой (основной):**
1. Главная → карточка своей проблемы (экран 2)
2. Страница-решение (диалог с госорганами / судебная практика / третейский
   суд / эксперты / международка) — на каждой есть кейсы и CTA
3. Не готов платить → подписка на дайджест или вопрос через контакты
   (мы получили контакт — «прогреваем»)
4. Готов → заявка на вступление (4 шага, 2 минуты) → звонок в течение 24 ч
   → одобрение Правления → взнос

**Флоу Б — поиск экспертизы (страница «Эксперты»):**
1. Подбор эксперта: шаг 1 — отрасль, шаг 2 — тема вопроса
2. Карточки подходящих экспертов с компетенциями → «Задать вопрос»
3. Если именного эксперта нет — не тупик, а форма: «дирекция и сеть
   Клуба 50+ подберут специалиста за 24 часа» (каждый «промах» = лид)
4. Блок «Компетенции союза» складывается автоматически из навыков всех
   экспертов — витрина экспертности, подтверждённая именами

**Флоу В — навигация (5 пунктов вместо 35 ссылок):**
- **Чем помогаем** — решения проблем + аналитика
- **Результаты** — кейсы, советы, награды, СМИ
- **Членство** — преимущества, виды и цены, вступить
- **О союзе** — миссия, история, руководство, ассоциации
- **Контакты**

Второстепенное (Кунявский, символика, регламент, иностранным инвесторам)
живёт в футере — доступно, но не отвлекает.

## 4. Этапы: от демо к работающему ресурсу

### Этап 0 — Демо (готово)
- Новая воронка главной, упрощённая навигация, каталог экспертов с подбором
- Убраны разделы-заглушки из меню, фейковые новости, чужие логотипы
- Цифры сведены к единым (94 компании, 20+ советов, 35 лет, 7 ассоциаций)
- Реальные имена дирекции на странице руководства и в экспертах

### Этап 1 — Контент от союза (1–2 недели, критический путь)
Блоки, которые заполняются **сразу** — без разработки, только информация:
1. **Подтвердить цифры**: точное число членов, отрасли, суммы взносов по видам членства
2. **Эксперты — 10–15 карточек**: дирекция (2 готовы) + Клуб 50+ + руководители
   якорных членских организаций. По каждому: фото, роль, отрасли, компетенции,
   3–5 строк фактов
3. **Кейсы — 6–8 штук** из квартальных отчётов в формате
   «Проблема → Что сделали → Результат» (3 уже на главной)
4. **Согласия членов** на упоминание/логотипы (тогда отрасли можно заменить именами)
5. **Фото**: руководство, офис, мероприятия

Блоки, которые **вытекают** из этапа 1:
- страница «Достижения» наполняется кейсами;
- «Компетенции союза» растут автоматически с каждым новым экспертом;
- каталог членов (для членов) наполняется после согласий.

### Этап 2 — Технический запуск (параллельно, ~1 неделя)
1. Выполнить SQL для подписчиков дайджеста (уже в `supabase/schema.sql`,
   таблица `digest_subscribers`)
2. Перенести экспертов из кода в базу (структура готова 1:1)
3. Домен, аналитика (Метрика/GA), проверить формы заявок
4. Назначить ответственного за обработку заявок и вопросов (SLA 24 часа —
   это обещание на сайте)

### Этап 3 — Регулярный процесс (постоянно, главное условие «живого» сайта)
1. **Квартальный отчёт → 3–5 кейсов на сайт** (отчёт уже пишется — только переупаковка)
2. **Правовой дайджест раз в 2 недели** (правовые обзоры уже готовятся для
   членов — открытая версия становится лид-магнитом)
3. Ведение Telegram-канала синхронно с сайтом
4. Только после наладки процесса — включать разделы «Новости» и «Мероприятия»
   (сейчас скрыты, чтобы сайт не выглядел заброшенным)

### Этап 4 — Развитие (после запуска, по стратегии союза)
- Личный кабинет члена: документы, консультации, счета (каркас уже в коде)
- CRM-процесс: подписчик дайджеста → участник мероприятия → член союза
- Страницы экспертов как SEO-посадочные («эксперт по трудовому праву Беларусь»)
- PDF-гайд «Почему БСПН нельзя заменить» как скачиваемый лид-магнит

## 5. Ключевые тезисы для Правления

1. Сайт продаёт **результат**, а не организацию: боли → решения → доказательства → цена.
2. У посетителя всегда есть **лёгкий следующий шаг** (дайджест, вопрос), а не только «заплати взнос».
3. Контент уже существует — квартальные отчёты и правовые обзоры. Нужен только
   процесс переупаковки, а не новые ресурсы.
4. Экспертность показывается **именами и навыками**, а сумма навыков экспертов
   и есть компетенция союза — это напрямую реализует цель стратегии
   «центр компетенций и экспертизы».
5. Что нужно от Правления: подтверждение цифр, 10–15 кандидатов в эксперты,
   согласия членов, ответственный за заявки.
